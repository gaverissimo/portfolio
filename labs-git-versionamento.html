Guia rápido: Versionamento com Git e GitHub

Procedimento básico para versionamento local com Git e publicação de projetos no GitHub.

Versionamento com Git e GitHub

Inicializar repositório local

1. Abrir o terminal na pasta do projeto

2. Inicializar o Git

                git init

Função: Cria a pasta oculta .git, responsável por armazenar:

  • Histórico de alterações
  • Commits
  • Versões do projeto

Verificar status dos arquivos

                git status

Função: Mostra arquivos modificados ou ainda não versionados.

Adicionar arquivos para commit

                git add .

Função: Envia os arquivos para a staging area, preparando para o commit.

Criar um commit

                git commit -m "Primeira versão do portfolio"

Função: Salva um snapshot do projeto naquele momento.

Criar repositório no GitHub

  1. Clicar em New repository
  2. Definir um nome
  3. Selecionar Public
  4. Não marcar a opção README
  5. Clicar em Create repository

Conectar projeto local ao GitHub

                git remote add origin https://github.com/SEU-USUARIO/portfolio.git
                git branch -M main
                git push -u origin main

Função: Envia o projeto local para o repositório remoto.

Fluxo resumido

                git add .
                git commit -m "mensagem"
                git push

Como funciona o Git no projeto?

                Versiona o codigo localmente com Git, faz commits das alteracoes e envia para o GitHub usando push. 
                Isso permite manter historico e trabalhar com CI/CD depois.
