Docker básico – Executando um site com Nginx

Guia rápido com comandos básicos para executar um site simples utilizando Docker e Nginx.

Fluxo do ambiente

Computador (host) → Container Docker → Nginx → Site estático

Conceitos básicos

Docker

Ferramenta que permite empacotar aplicações e suas dependências em containers, garantindo que rodem de forma consistente em qualquer ambiente.

Container

Ambiente isolado onde a aplicação é executada junto com todas as dependências necessárias.

Nginx

Servidor web utilizado para servir páginas estáticas ou atuar como proxy reverso.

Comandos básicos

Ver containers em execução

docker ps

Parar um container

docker stop ID_DO_CONTAINER

Remover um container

docker rm ID_DO_CONTAINER

Listar todos os containers (inclusive parados)

docker ps -a

Executando Nginx apontando para o site

Comando:

docker run -d -p 3000:80 \
-v $(pwd):/usr/share/nginx/html \
nginx

O que esse comando faz

  • docker run → cria e inicia um container
  • -d → executa o container em segundo plano
  • -p 3000:80 → mapeia a porta 3000 do computador para a porta 80 do container
  • -v $(pwd):/usr/share/nginx/html → monta a pasta atual dentro do container
  • nginx → imagem utilizada para criar o container

O Docker utiliza a pasta atual como diretório do site dentro do Nginx:

/usr/share/nginx/html

Resultado

Após executar o comando, o site pode ser acessado no navegador:

http://localhost:3000

Nesse exemplo foi utilizada a porta 3000. Pode ser utilizada qualquer porta disponível em seu ambiente.

Explicação

Containerização de um site estático utilizando Docker e Nginx, mapeando o diretório local para o diretório padrão do servidor web dentro do container e expondo a porta para acesso pelo navegador.
